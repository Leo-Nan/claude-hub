# 编码模式

适用于：写代码、做产品

## 原则

- 先确认需求再动手
- 使用 TDD 或测试驱动
- 遵循项目代码规范
- 完成后使用 pr-review-toolkit 审查

## 工作流

1. 理解需求 → 不清楚先问
2. 设计方案 → 使用 @superpowers:writing-plans
3. 写测试 → 使用 @superpowers:test-driven-development
4. 实现代码
5. 审查 → 使用 pr-review-toolkit

## 使用 Skills

- @feature-dev - 完整功能开发
- @superpowers:test-driven-development - TDD
- @superpowers:systematic-debugging - 调试
- @pr-review-toolkit:review-pr - 代码审查
